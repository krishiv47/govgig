# GovGig Portal Hackathon Presentation

## 1. Title
- GovGig Portal
- Government-backed registry and operations portal for gig workers
- Focus: verified workers, company sourcing, compliance, and insurance access

## 2. Problem
- Gig workers often operate without structured protections
- Companies need verified workers but access is fragmented
- Working hours, fuel, and traffic compliance are poorly tracked
- Insurance access is limited or disconnected from actual work records

## 3. Solution
- A government registry for verified gig workers
- Company access based on approved worker data sharing
- Traffic-aware speed and working-hours compliance tracking
- Insurance marketplace with plans posted by different companies

## 4. Working Model
- Idle map shows worker location with nearby pickup and drop preview
- Company request assignment changes the route
- Worker marker moves across the map live
- Fuel, speed, ETA, and remaining distance change continuously

## 5. Key Features
- Worker registry and identity view
- Company demand board
- Traffic-aware compliance monitoring
- Working-hours tracker
- Anonymous worker ratings
- Health insurance scheme browsing and buying

## 6. Why It Matters
- Better worker protection
- Better worker verification
- Better compliance visibility
- Better access to insurance and welfare benefits
- Better company access to verified gig workers

## 7. Tech Stack
- HTML
- Vanilla JavaScript
- Leaflet
- OpenStreetMap

## 8. Future Scope
- Government API integration
- Real worker KYC and verification
- Live traffic and telematics input
- Real company-side sourcing workflow
- Real insurance onboarding and claims

## 9. Closing
- GovGig Portal connects worker protection, company demand, and insurance access in one system
